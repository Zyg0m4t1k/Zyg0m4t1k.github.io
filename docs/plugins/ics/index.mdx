# Description

Ce plugin permet de récupérer des calendriers distants (Apple, Google, Nextcloud, CalDAV…) ou locaux.  
Il est possible d’interagir avec ces calendriers et de lancer des actions selon les événements.

---

# Configuration du plugin

Après le téléchargement du plugin, il vous suffit de l’activer et de le configurer.

:::warning IMPORTANT
Ce plugin **ne permet pas de modifier ou éditer un calendrier distant**.
:::

En mode **Debug**, le plugin est très verbeux.  
Il est recommandé d'utiliser ce mode seulement si vous devez diagnostiquer un problème particulier.

Il n’est **pas recommandé de laisser le plugin en Debug en permanence**, surtout si Jeedom fonctionne sur **SD-Card**.

Une fois le diagnostic terminé, repassez le niveau de log sur **Error**.

:::note
Le plugin permet de :

- récupérer un calendrier via URL (Apple, Google, Nextcloud, Synology, CalDAV) et d’interagir en créant des événements
- récupérer un calendrier **ICS local** et l’intégrer à Jeedom
- créer des événements liés à un calendrier avec le même nom, et déclencher des actions au début ou à la fin de l’événement  
  (donc en créant un événement sur votre calendrier iOS ou Google, cela peut lancer des actions dans Jeedom)
- avoir un **widget dashboard paramétrable**
:::

![configuration](/img/plugins/ics/ics_conf.png)

**?title?** : retourne le titre de l'événement  
**?hour?** : retourne l'heure de début de l'événement

Il faut configurer les champs pour le retour des informations avec les commandes **today** et **tomorrow** d'un calendrier  
(commande à utiliser ensuite via TTS, notification...). Mettre `?title?` pour le titre de l'événement et `?hour?` pour l'heure.

# Les évènements

:::note
Tout l'intérêt de ce plugin réside dans la gestion des événements.
:::

:::warning IMPORTANT
Le plugin est unidirectionnel et récupère seulement les événements d'un calendrier (iOS, Google, Nextcloud, CalDAV…).  
Aucune modification ne peut être effectuée.
:::

---

## Important — Événements All-day (journée entière)

Les événements **All-day** (journée entière) ont un comportement spécifique :

- **Une seule action “Début”** est exécutée, **à l’heure configurée** dans *Heure actions (All-day)* (ou **07:00** par défaut).
- Il n’y a **pas d’action de fin** pour les All-day (même si l’événement se termine à 00:00 le lendemain).
- Un All-day **sur plusieurs jours** ne déclenche **pas tous les jours** : l’action “Début” est exécutée **une seule fois**, au début de la période.

Pour filtrer précisément les événements (titre/description), voir : **Reconnaissance des événements**.

---

:::tip
L'avantage dans ce cas est de nommer le calendrier avec un mot clé (anniversaire, rdv, etc.) et de pouvoir être averti si cet événement a lieu aujourd'hui ou demain (voir les commandes par la suite).
:::

![ics9](/img/plugins/ics/ics9.png)

Paramètres :

- **Calendrier** : permet de choisir le calendrier pour lequel l’événement est actif
- **Variable** : crée une variable si remplie, utilisable ensuite dans les scénarios / actions
- **Exceptés Fériés** : si coché, il faut choisir un calendrier défini comme férié (voir infos sur les calendriers). Les événements ne se déclencheront pas dans ce cas.

## Reconnaissance des événements

Le plugin peut matcher des événements via :

- le **titre**
- la **description**
- des **regex** (si activé)

Les **regex** sont supportées : tu peux mixer mots-clés simples et expressions régulières (format `/.../i`).

### Exemples de configuration (zone de droite)

![Configuration](/img/plugins/ics/config_regex.png)

La zone de configuration (voir capture) permet notamment :

- **Calendrier** : sélectionner la source (Google / iCloud / CalDAV…)
- **Variable** : optionnel, pour exposer un état/valeur dans Jeedom (selon tes usages)
- **Exceptés Fériés** : ignorer les jours fériés (si configuré)
- **Règles RegExp** : filtrage avancé (une règle par ligne, ou regex au format `/pattern/i`)
- **Insensible à la casse** : active automatiquement le flag `i` si tu ne fournis pas `/.../i`
- **Inclure Description** : inclure la description dans la recherche/matching
- **Heure actions (All-day)** : heure de déclenchement pour les événements “journée entière”
- **Test RegExp** : tester rapidement sur titre / description



### Fonctionnement des règles RegExp

Les règles RegExp permettent de filtrer les événements du calendrier pour déclencher une action dans Jeedom.

Le plugin analyse :

• **le titre uniquement** si *Inclure Description* est désactivé  
• **le titre + la description** si *Inclure Description* est activé  

Chaque règle peut être :

• **un mot simple**

exemple :

anniversaire

• **une expression régulière**

exemple :

/anniv(er)?saire/i  
/rdv.*dentiste/i

Le `i` signifie que la recherche **ignore les majuscules/minuscules**.

Si **plusieurs règles sont définies**, une seule correspondance suffit pour déclencher l'événement.

Le bouton **Test RegExp** permet de vérifier rapidement si une règle correspond au titre ou à la description d’un événement.

# Les commandes

## Calendriers

![ics11](/img/plugins/ics/ics11.png)

- **Last Event** : permet de récupérer le dernier événement d’un calendrier qui a déclenché les actions
- **Is holiday today** (pour un calendrier défini comme férié) : retourne 1 si jour férié, sinon 0
- **Is holiday tomorrow** (pour un calendrier défini comme férié) : retourne 1 si le lendemain est férié, sinon 0
- **Today** : récupère tous les événements de la journée (TTS, mail…)
- **Tomorrow** : récupère tous les événements du lendemain (TTS, mail…)
- **Next** : le prochain événement à venir
- **Event Today** : retourne 1 si événement le jour même, sinon 0
- **Event Tomorrow** : retourne 1 si présence d'événement le lendemain, sinon 0
- **heure** : retourne l'heure du prochain événement
- **titre** : retourne le titre du prochain événement
- **date** : retourne la date du prochain événement
- **togo** : retourne 1 si événement encore le jour même à venir, sinon 0

## Evènements

![ics12](/img/plugins/ics/ics12.png)

- **Today** : retourne 1 si l’événement a lieu le jour même, sinon 0
- **Tomorrow** : retourne 1 si l’événement a lieu le lendemain, sinon 0
- **Description** : retourne la description de l'évènement en cours si elle existe
- **Description (Next)** : retourne la description du prochain évènement si elle existe
- **heure** : retourne l'heure de la prochaine occurrence si elle existe
- **titre** : retourne le titre de la prochaine occurrence si elle existe
- **date** : retourne la date de la prochaine occurrence si elle existe
- **Localisation** : retourne la localisation de l'évènement si elle existe
- **Now** : retourne 1 si l'évènement est en cours, sinon 0
- **Next** : retourne le timestamp de l'évènement s'il est à venir ou 0 (défini selon le nombre de jours sauvegardés)

## Utilisation des variables

Je crée un événement nommé **Anniversaire** pour le calendrier **Caldav**.  
Un événement **Anniversaire de Paul** existe dans ce calendrier.

Je vais créer une variable **anniversaire** pour l'événement **Anniversaire** et enregistrer.

![ics16](/img/plugins/ics/ics16.png)

Ensuite dans les actions de début d'événement je donne comme valeur à la variable le nom de l'événement en utilisant la commande **Last event** du calendrier **Caldav**.  
Il est aussi possible de donner la valeur de la description de l'événement.

![ics17](/img/plugins/ics/ics17.png)

Ensuite, dans un scénario, vous pouvez récupérer cette valeur de plusieurs façons :

- mettre la variable comme déclencheur
- programmer un scénario qui retourne la variable via une interaction

## Scénario utilisant les informations des calendriers

On utilise donc les commandes des calendriers et on définit des actions selon :

- l'heure du prochain événement
- le titre
- etc.

![scenario1](/img/plugins/ics/scenario1.png)

**Déclencheur** : on peut utiliser la commande **Next event** du calendrier qui se lancera lors de la mise à jour du prochain événement.

**Premier bloc if** : `#[Plugins][Famille][A venir]# == 1` vérifie que l'événement a lieu le jour même.

**Bloc A** : `#[Plugins][Famille][Heure]#` retourne l'heure de l'événement à venir.  
On peut utiliser `time_op(#[Plugins][Famille][Heure]#,+20)` par exemple.

**Deuxième bloc if** : `#[Plugins][Famille][Titre]# == "Boulot Zyg0"` vérifie que le titre est correct et si OK lance les actions.

:::note
C'est un exemple et c'est paramétrable à souhait.  
Faire de même avec un scénario utilisant les informations des événements en mettant la variable de l'événement en déclencheur par exemple.
:::
