Le plugin **Telecommandes** permet de créer des **télécommandes virtuelles entièrement personnalisables** dans Jeedom.

Chaque bouton peut déclencher :

- une **commande Jeedom** (action, info)
- un **scénario**
- une **interaction**
- toute **action compatible Jeedom**

Cela permet de regrouper plusieurs actions dans une **interface rapide et visuelle**, idéale pour un dashboard, un design ou un écran tactile mural.

---

## ✨ Principe de fonctionnement

Telco crée des équipements Jeedom dont chaque **bouton est associé à une action**.

Appuyer sur un bouton → exécuter l'action configurée.

Exemples d'usage :

- Télécommande **multimédia** pour un salon (TV, box, son)
- Télécommande **maison** (volets, lumières, alarme)
- Panneau de contrôle **scénarios favoris**
- Interface simplifiée pour un **écran tactile**
- Télécommande accessible pour un **utilisateur non technique**

---

## 🔧 Prérequis

- Jeedom fonctionnel (version 4.4+ recommandée)
- Des commandes ou scénarios déjà créés dans Jeedom à associer aux boutons

## 🧩 Les deux types de télécommandes

### 1️⃣ Télécommande Multimédia

Le mode **Multimédia** génère automatiquement une télécommande standard avec un layout pré-défini adapté à la commande d'équipements audiovisuels.

Elle comprend les boutons classiques :

- Power On / Off
- Navigation (haut, bas, gauche, droite, OK)
- Volume + / - / Mute
- Chaînes + / -
- Lecture / Pause / Stop / Suivant / Précédent
- Boutons de couleur (Rouge, Vert, Jaune, Bleu)
- Touches numériques 0–9
- Menu, Retour, Accueil

Chaque bouton peut être **personnalisé** : vous lui associez la commande Jeedom de votre choix.

### 2️⃣ Télécommande Personnalisée (Custom)

Le mode **Custom** vous permet de créer vos propres boutons de zéro.

Pour chaque bouton, vous définissez :

| Paramètre | Description |
|---|---|
| **Nom** | Libellé affiché sur le bouton |
| **Icône** | Icône FontAwesome associée |
| **Couleur** | Couleur du bouton |
| **Action** | Commande ou scénario Jeedom à exécuter |
| **Visible** | Afficher ou masquer le bouton |

Cela permet de créer des interfaces **totalement sur-mesure** sans contrainte de layout.

## ⚙️ Configuration d'un bouton

Dans la page de configuration de l'équipement, chaque bouton dispose de ses propres réglages.

### Associer une action à un bouton

1. Cliquez sur le bouton à configurer
2. Choisissez la **commande Jeedom** cible (via le sélecteur)
3. Ou choisissez un **scénario** à déclencher
4. Sauvegardez

<!-- screenshot: configuration d'un bouton avec le sélecteur de commande -->
<!-- path: /img/plugins/telco/button_config.png -->
<!-- ![Configuration d'un bouton avec association de commande](/img/plugins/telco/button_config.png) -->

### Désactiver un bouton

Un bouton peut être **désactivé** (masqué sur le widget) sans être supprimé — utile pour nettoyer l'interface sans perdre la configuration.

## 💡 Exemples d'usage concrets

### Télécommande TV / Box

Contrôle d'un téléviseur Android TV ou d'une box FAI :

- Power → `[TV][Allumer]`
- Volume + → `[TV][Volume+]`
- Netflix → `[TV][Lancer Netflix]`
- Chaîne préférée → `[Box][Chaîne TF1]`

## 🧠 Bonnes pratiques

- Utilisez des **icônes FontAwesome explicites** pour que les boutons soient compréhensibles sans lecture du label
- Regroupez les boutons par **thématique** dans des télécommandes séparées (audio, lumières, scénarios)
- Sur un **écran tactile**, privilégiez des boutons larges avec peu de texte
- Utilisez la **télécommande personnalisée** pour des interfaces accessibles à des utilisateurs non techniques
- Testez chaque bouton depuis la page de configuration avant de l'exposer sur le dashboard

---

## 🔍 Dépannage

### Un bouton ne déclenche rien

- Vérifiez que la commande associée est bien **de type action** et non info
- Vérifiez que l'équipement cible est **activé** dans Jeedom
- Testez la commande directement depuis sa page dans Jeedom

### Le widget n'apparaît pas sur le dashboard

- Vérifiez que l'équipement est **visible** et **activé**
- Vérifiez qu'il est bien associé à un **objet parent** affiché sur le dashboard
