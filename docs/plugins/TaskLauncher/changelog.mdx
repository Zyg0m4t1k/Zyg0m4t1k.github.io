## 28.04.2026

### Nouveautés
- Nouveau widget **"Réveil Analogique"** (`reveil_analog`) : horloge analogique CSS/JS pure avec 4 aiguilles (heures, minutes, secondes, alarme), pistes circulaires, dots lumineux et effet dôme 3D.
- Support complet des **thèmes Jeedom Light et Dark** pour tous les widgets via un fichier utilitaire CSS dédié (`jeedom-theme-utils.css`) basé sur les variables CSS natives de Jeedom.

### Améliorations
- Widget **"Design Réveil"** (`reveil_design`) : les boutons H−/H+ et M−/M+ sont maintenant regroupés en une seule ligne sous le cercle SVG.
- Widget **"Réveil Analogique"** : suppression des boutons de jours sur le widget (les jours restent configurables et enregistrés, l'affichage est épuré).
- Adaptation automatique des couleurs, fonds, bordures et ombres selon le thème actif (Light/Dark) pour les widgets réveil, design et événement.

## 06.03.2026

### Nouveautés
- Ajout d’une **vue panel desktop** pour afficher les équipements TaskLauncher par objet Jeedom.
- Ajout d’une **vue panel mobile** avec navigation par objet.
- Ajout d’un **modal de création rapide** d’équipement (Réveil / Horloge / Événement).
- Ajout d’un **thème CSS dédié** pour les widgets (`tasklauncher-theme.css`).

### Améliorations
- Refonte importante de l’interface d’administration du plugin.
- Modernisation du JavaScript (gestion des événements `.on()` / `.off()`).
- Amélioration du `datetimepicker` et de la gestion des champs date/heure.
- Réorganisation et nettoyage du code PHP et JS.
- Meilleure gestion du chargement des équipements par objet.
- Réorganisation des traductions et alignement avec les langues déclarées dans `info.json`.

### Correctifs
- Nettoyage renforcé des **crons liés aux équipements** lors de la suppression d’un eqLogic.
- Suppression automatique des crons `TaskLauncher::launchEventAlarm` lors de la désinstallation du plugin.
- Correction de plusieurs problèmes JS (`.value()` remplacé par `.val()`).
- Suppression de logs debug inutiles.
- Amélioration de la stabilité de l’interface mobile et dashboard.

### Packaging
- Suppression de la documentation embarquée dans le plugin.
- Utilisation de la documentation et du changelog **hébergés en ligne**.
- Mise à jour du manifeste `info.json` :
  - compatibilité **Jeedom 4.4+**
  - support **Debian 11 → 13**
  - description multilingue.
