:::caution ⚠️ Informations importantes

📡 **Portée Bluetooth**

La portée BLE dépend fortement :

- de la distance
- des murs
- des interférences radio

➡️ Une **antenne Bluetooth externe** peut améliorer la réception.

🔵 **Conflit Bluetooth**

Si d'autres plugins utilisent le Bluetooth (BLE), des conflits peuvent apparaître lors du scan.

➡️ Il est recommandé de **stopper les démons BLE des autres plugins pendant un scan**.

🔁 **Synchronisation d'état**

Si l’équipement est utilisé directement via l’application **SwitchBot**, l’état peut ne pas être immédiatement synchronisé avec Jeedom en mode APi unique.

:::

# 📦 Présentation

Le plugin **SwitchBot** permet d'intégrer les équipements **SwitchBot** dans **Jeedom**.

Il supporte deux méthodes de communication :

- ☁️ **API Cloud SwitchBot**
- 📡 **Bluetooth Low Energy (BLE)**

Le plugin permet notamment :

- création automatique des équipements
- pilotage des appareils
- récupération des états
- utilisation dans les **scénarios Jeedom**
- supervision via la **page santé**

---

# 🧰 Équipements supportés

## ☁️ Équipements via API

Ces équipements nécessitent un **Hub SwitchBot**.

Compatibles :

- SwitchBot Bot
- SwitchBot Curtain
- SwitchBot Curtain 3
- SwitchBot Lock
- SwitchBot Meter
- SwitchBot Meter Plus
- SwitchBot Outdoor Meter
- SwitchBot Motion Sensor
- SwitchBot Contact Sensor
- SwitchBot Color Bulb

Fonctions :

- récupération des états
- pilotage des équipements
- synchronisation avec le cloud SwitchBot

---

## 📡 Équipements via BLE

Le plugin peut communiquer **directement en Bluetooth**.

Compatibles :

- SwitchBot Bot
- SwitchBot Meter
- SwitchBot Meter Plus
- SwitchBot Motion Sensor
- SwitchBot Contact Sensor
- SwitchBot Curtain
- SwitchBot Curtain 3

Avantages :

⚡ retour d'état quasi instantané  
🏠 fonctionnement local  
🚀 latence réduite  

---

# ⚙️ Modes de fonctionnement

Le plugin propose **trois modes de communication**.

| Mode | Description |
|-----|-----|
| ☁️ API | Toutes les communications passent par le cloud SwitchBot |
| 📡 BLE | Communication locale via Bluetooth |
| 🔄 Auto | Combine BLE pour les états et API pour les actions |

---

## ☁️ Mode API

Toutes les communications passent par **l'API SwitchBot**.

### Avantages

✔ configuration simple  
✔ pas besoin de Bluetooth  

### Inconvénients

❗ latence plus élevée  
❗ dépendance au cloud

---

## 📡 Mode BLE

Communication directe via Bluetooth.

Un **daemon Python** gère :

- le scan BLE
- la réception des données
- la mise à jour des états

### Avantages

✔ communication locale  
✔ retour d'état rapide  
✔ indépendance du cloud

---

## 🔄 Mode Auto

Mode hybride :

- états via **BLE**
- actions via **API**

Ce mode offre généralement :

✔ meilleure fiabilité  
✔ compatibilité maximale

---

## 📱 Procédure

### 1️⃣ Ouvrir l'application

Dans l'application **SwitchBot** :

- ouvrir l'app
- aller dans **Profile / Profil**

---

### 2️⃣ Activer les options développeur

Aller dans :

**Preferences / Préférences**

Puis :

- descendre tout en bas
- cliquer sur **App Version**

⚠️ **Appuyer 10 fois rapidement sur "App Version"**

Une nouvelle section apparaît :

➡️ **Developer Options**

---

### 3️⃣ Récupérer les clés

Dans **Developer Options** :

- copier le **Token**
- copier le **Secret**

Puis les renseigner dans la **configuration du plugin SwitchBot**.

---

:::tip
Les clés API sont liées à votre compte SwitchBot.

Si vous les modifiez dans l'application, pensez à **mettre à jour la configuration dans Jeedom**.
:::

# 🔧 Dépannage

## BLE ne détecte aucun équipement

Vérifier :

- l'adaptateur Bluetooth
- que le daemon est actif
- qu'aucun autre plugin BLE n'utilise l'adaptateur

---

## API ne fonctionne pas

Vérifier :

- Token
- Secret
- connexion Internet
