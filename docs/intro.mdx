## 👋 Bienvenue

Ici tu trouveras la documentation de **mes plugins Jeedom** : des pages courtes, structurées, et orientées “résultat”.
Le but : aller vite, éviter les pièges classiques, et avoir un endroit unique où tout est à jour.

---

## ✅ Ce que tu trouveras sur ce site

- **Guides d’installation** : prérequis, dépendances, permissions, venv (quand nécessaire)
- **Configuration pas à pas** : écrans Jeedom + exemples concrets
- **Commandes & scénarios** : comment exploiter le plugin dans la vraie vie
- **Dépannage** : erreurs fréquentes, quoi vérifier en priorité, où regarder
- **Compatibilité** : Jeedom / OS / PHP min, migrations, notes importantes

## 🚀 Démarrer en 2 minutes

### 1) Choisir ton plugin
Dans le menu **Plugins** (à gauche), ouvre la page du plugin qui t’intéresse.


### 2) Installer puis configurer
Suis l’ordre des sections :
1. Installation
2. Configuration
3. Fonctionnement / cas d’usage

### 3) En cas de problème : logs d’abord
Avant de chercher plus loin, récupère dans Jeedom :
- **Log du plugin**
- **http.error** (si souci UI / AJAX)
- **daemon** (si le plugin utilise un service Python)

---

## 🧩 Bonnes pratiques (rapides)

- Toujours vérifier la **version Jeedom** et la **version du plugin**
- Après un update, faire un **redémarrage du démon** (si applicable)
- Si un équipement “ne bouge pas”, vérifier que les **commandes** remontent bien et que les **dépendances** sont OK

---

## 🐞 Remonter un bug / demander une amélioration

Pour gagner du temps (et éviter les aller-retours), merci d’indiquer :

- Version **Jeedom** + **OS**
- Version du **plugin** (et branche : stable/beta/dev)
- Extrait de log (sans infos sensibles)
- Étapes pour reproduire + ce que tu attends
